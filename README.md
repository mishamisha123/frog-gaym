# Froggy Leap Deluxe

A mobile-first, installable HTML5 game for GitHub Pages. It uses only HTML, CSS, JavaScript, Canvas, Web Audio, and generated PNG icons—there are no external libraries or paid assets.

## Included gameplay

- Start with **1,000 Froggy** virtual currency.
- Choose bets of 50, 100, 250, 500, or MAX.
- The frog crouches, launches in an arc, spins, and squash-stretches on landing.
- Ten increasingly risky jumps lead from **1.10×** to **12.00×**.
- Cash out after any successful jump, or risk the next lily pad.
- Failed pads crack, sink, splash, shake the screen, and end the round.
- Procedural water, particles, ripples, glows, confetti, and generated sound effects.
- XP, levels, statistics, virtual balance saving, frog skins, and seven lake themes.
- Daily reward wheel, streaks, lucky charms, and guaranteed skin rewards on eligible 7-day streaks.
- Installable PWA with offline caching after the first successful visit.
- Responsive support for iPhone, Android, tablets, and desktop browsers.

Froggy is fictional in-game currency only. The project contains no payment system and offers no real-money prizes.

## Publish on GitHub Pages

1. Upload **all files and folders in this package** to the root of your repository. `index.html` must be visible at the repository root.
2. Open the repository's **Settings**.
3. Open **Pages** under “Code and automation.”
4. Set **Source** to “Deploy from a branch.”
5. Select the `main` branch and `/ (root)`, then save.
6. Wait for GitHub Pages to show the published address.

The address will normally look like:

`https://YOUR-USERNAME.github.io/YOUR-REPOSITORY/`

## Install on iPhone

1. Open the GitHub Pages address in Safari.
2. Tap Safari's Share button.
3. Choose **Add to Home Screen**.
4. Launch Froggy Leap from its new icon.

## Updating

Replace the changed files in the repository and commit them. GitHub Pages redeploys automatically. If an installed copy temporarily shows the old version, close it completely and reopen it while connected to the internet so the service worker can update.
